# Magic Diary

Magic Diary 是一张会读取手写内容并以墨迹回信的数字日记页。它以纸张、书写和回信为主要交互语言，而不是聊天界面。

## Language

**回复模型**:
生成日记最终回信的模型；它可能支持直接读取图片，也可能只能处理文字。
_Avoid_: 主模型、聊天模型

**识字模型**:
在回复模型无法直接读取图片时，将手写墨迹转写为文字的视觉模型。
_Avoid_: 辅助模型、补充视觉模型、OCR 模型

**直读能力**:
回复模型直接从手写图片理解内容并生成回信的能力。
_Avoid_: 单模型模式、自带视觉

**自动能力路由**:
系统根据回复模型的直读能力，选择直接回信或先由识字模型转写；未知能力可通过内置图片测试确认，并允许用户覆盖。
_Avoid_: 自动模式、智能模式

**内置体验**:
由日记服务在后端预设并承担额度的模型服务；用户无需配置，也不会收到上游地址、密钥或具体模型信息。
_Avoid_: 默认模型、免费模型、开发者模型

**自带 API**:
用户主动提供并自行承担额度的模型服务；其地址、密钥和模型选择属于该用户的本地配置。
_Avoid_: 自定义模型、外部模型、私人模型

**自动中转**:
自带 API 无法由浏览器直接连接时，日记服务临时转发该请求；密钥仍只持久化在用户浏览器，但会经过中转且不被记录。
_Avoid_: 代理模式、隐私代理、后端连接

**测试资格**:
邀请码授予的内置体验使用资格，带有独立的每日额度；首次启用需要完成人机验证。
_Avoid_: 用户账号、免费额度、白名单

**使用额度**:
内置体验中新书写回合成功获得回信的次数；失败、能力回退和重读不消耗额度，同一份墨迹最多重读三次。
_Avoid_: 请求次数、模型调用次数、token 额度

**重读**:
用户划掉一封理解有误的回信后，日记用识字模型重新辨认同一份墨迹，再生成新的回信。
_Avoid_: 重试、重新生成、纠错

**首笔**:
回信流中第一批可见墨迹开始写到纸面的时刻；完整标点、累计八个汉字或首字符到达六百毫秒后，任一条件满足即可落笔。
_Avoid_: 首屏、首次响应

**回信锚点**:
回信开始书写的位置，默认是用户首行首字的左上角；仅在安全区或可用空间不足时做最小幅度修正。
_Avoid_: 回复位置、跟随位置、自动居中

**固定居中**:
一种高级回信锚点选项，将回信放在纸面安全区域的视觉中心；默认不开启。
_Avoid_: 默认位置、自动位置

**自适应墨迹图**:
由本轮全部用户笔画及动态留白组成、实际发送给模型的图像；不包含回信、界面文字、纸张纹理或系统界面。
_Avoid_: 局部截图、屏幕截图、整页截图

**当前页上下文**:
当前日记页内连续书写和回信形成的对话；默认保持连贯，并在新建页面时结束。
_Avoid_: 当前会话、聊天记录

**日记存档**:
经用户明确开启后，保存在当前浏览器中的过去页面内容。
_Avoid_: 历史记录、聊天历史

**长期记忆**:
经用户单独开启后，从日记存档中选取与当前书写相关的内容，作为跨页回信依据。
_Avoid_: 历史上下文、永久记忆

**上下文预算**:
当前页上下文保持原文的软上限；默认约八千 tokens，超过后摘要较早内容并保留最近十轮原文。
_Avoid_: 轮数限制、上下文窗口

**吸收**:
日记在停笔约一点二秒后锁定本轮墨迹，并用约九百毫秒让墨迹融入纸面；模型请求与动画同时进行，不额外延后回信。
_Avoid_: 消失动画、提交动画、加载动画

**回应期限**:
模型请求发出后二点五秒仍无首笔时给出纸面反馈，五秒仍无首笔时停止请求并恢复用户墨迹；超时不自动发起另一轮请求。
_Avoid_: 请求超时、加载超时

**候选笔画**:
回信期间检测到、但尚未确认是用户书写的短暂落笔；形成足够路径、持续移动或紧接第二笔后才成为有效笔画并打断回信。
_Avoid_: 误触检测、延迟笔画

**回卷**:
有效落笔打断回信后，已写出的回信从最新采样点开始，按字符、笔画和路径的反向顺序快速收回纸面。
_Avoid_: 淡出、清除回复、倒放动画

**划除手势**:
用户直接在可见回信字形上连续涂划，表达对整封回信的否定并触发重读；意图由本地笔迹几何与字形命中判断，不调用模型。
_Avoid_: 双横线、AI 手势识别、删除笔画

**被打断回信**:
用户在回信书写期间开始新的有效书写；只有用户已经看见的完整句子进入当前页上下文，残句丢弃并标记回信未完成。
_Avoid_: 已取消回复、中断请求
