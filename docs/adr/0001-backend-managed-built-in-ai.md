# 后端托管内置体验

内置体验的上游地址、密钥、模型选择与能力路由只存在于后端，浏览器仅提交受限的预设标识和日记回合数据；用户也可以切换到自带 API。公开测试通过邀请码、人机验证、短期会话和每日额度控制成本，因为任何放入前端的预制密钥都可被提取，而开放通用代理会造成滥用风险。额度只在新的书写回合成功回信后扣除；失败、内部能力回退和重读不额外扣除，但同一墨迹最多重读三次并受请求频率限制。

自带 API 优先由浏览器直连；遇到明确的跨域限制时自动通过独立入口中转。用户密钥只持久化在其浏览器，但中转时会临时经过服务端，因此界面必须准确披露且服务端不得记录密钥、墨迹或回复；中转目标仅允许安全的 HTTPS 地址，并与内置体验的服务端密钥完全隔离。
